Poziomy logowania 📊

Kliknij poziom, żeby zobaczyć przykład!

🔍 DEBUG — szczegóły dla programisty
DEBUG Obliczam obrażenia: atak=15, obrona=8, wynik=7
ℹ️ INFO — zdarzenia w grze
INFO Gracz wszedł do lokacji: Ciemna Jaskinia
⚠️ WARN — uwaga, coś nie tak
WARN Zdrowie gracza niskie: 12/100 HP!
💥 ERROR — coś się zepsuło!
ERROR Nie można załadować mapy: FileNotFoundException

💡 DEBUG < INFO < WARN < ERROR — ustawiasz minimalny poziom, reszta jest filtrowana!

logback.xml ⚙️

Konfiguracja loggera — jeden plik rządzi wszystkim!

<!-- logback.xml -->
<configuration>
  <appender name="CONSOLE" class="...ConsoleAppender">
    <encoder>
      <pattern>%d{HH:mm:ss} %highlight(%-5level) %msg%n</pattern>
    </encoder>
  </appender>

  <root level="DEBUG">
    <appender-ref ref="CONSOLE" />
  </root>
</configuration>

Interfejsy funkcyjne 🧩

Java daje gotowe interfejsy na lambdy!

✅
Predicate<T>
"sprawdza prawdę"
Entity::isAlive
T → boolean
🍽️
Consumer<T>
"zjada wartość"
logger::info
T → void
🔄
Function<T,R>
"przekształca"
Item::getName
T → R

💡 filter() potrzebuje Predicate, forEach() potrzebuje Consumer, map() potrzebuje Function

// Wszystko się łączy!
monsters.stream()
  .filter(Entity::isAlive)    // Predicate
  .map(Entity::getName)     // Function
  .forEach(logger::info); // Consumer
